## Standard workflow

1. Confirm working directory and goal.
2. Select an approved OpenRouter alias, defaulting to `openrouter/fusion`.
3. Start OpenCode in the target project.
4. Reuse the existing project session when present.
5. Select the model alias with `/models`.
6. Start in Plan for non-trivial work.
7. Generate and validate the plan.
8. Switch to Build after plan acceptance.
9. Implement and verify.
10. Return to Plan whenever questions, ambiguity, or risk appears.
11. Repeat until the outcome is complete.

Do not ask the user to choose an AI provider during normal flow; the provider is OpenRouter.
