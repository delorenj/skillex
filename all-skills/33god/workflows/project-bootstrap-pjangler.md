# Workflow: Automated Project Bootstrap with pjangler

**Purpose**: Create, provision, and bootstrap new 33god projects using the integrated pjangler automation.

## Prerequisites

1. **Environment Setup**:
   - Plane API key configured in `$ZC/secrets.zsh`
   - pjangler installed and accessible on PATH
   - Empty target directory for new project

2. **Required Information**:
   - Project name (human-readable)
   - Project identifier (slug/key)
   - Project description

## Automated Bootstrap Process

### Step 1: Initialize Project Directory
Navigate to your empty project directory:
```bash
cd ~/code/your-new-project
```

### Step 2: Run pjangler Bootstrap
Execute the 33god initialization:
```bash
pjangler init 33god
```

### Step 3: Interactive Configuration
The command will prompt for:
1. **Project Name**: Human-readable name (e.g., "Revenue Dashboard")
2. **Identifier**: Unique slug (e.g., "revenue-dashboard")
3. **Description**: Brief project purpose

### Step 4: Automated Provisioning
pjangler automatically:
1. **Retrieves** Plane API key from `$ZC/secrets.zsh`
2. **Creates** Plane project in 33god workspace via API
3. **Generates** hydrated boilerplate:
   - `CLAUDE.md` - AI context documentation
   - `AGENTS.md` - Agent configuration
   - `.plane.json` - Project metadata with UUID
   - `_bmad/bmm/config.yaml` - BMAD configuration

## Generated Files Structure

### `.plane.json`
```json
{
  "project": {
    "id": "<auto-generated-uuid>",
    "name": "<your-project-name>",
    "identifier": "<your-identifier>",
    "description": "<your-description>",
    "workspace": "33god"
  }
}
```

### `CLAUDE.md`
Pre-configured with:
- Project context and purpose
- Architecture patterns
- Integration points with 33god ecosystem
- Event/command contracts placeholder

### `AGENTS.md`
Pre-configured with:
- Agent roles and responsibilities
- Capability declarations
- Workflow associations
- Event subscriptions

### `_bmad/bmm/config.yaml`
Pre-configured with:
- BMAD structure declarations
- Module boundaries
- Service configurations

## Post-Bootstrap Steps

### 1. Verify Plane Integration
```bash
# Check project was created in Plane
cat .plane.json | jq '.project.id'
```

### 2. Initialize Git Repository
```bash
git init
git add .
git commit -m "Initial 33god project scaffold via pjangler"
```

### 3. Define Initial Tickets
Create first 3 executable tickets in Plane:
1. Core domain model setup
2. Initial event contracts
3. First service implementation

### 4. Configure Event Contracts
Update `CLAUDE.md` with specific events/commands:
```markdown
## Event Contracts
- `ProjectInitialized` - Emitted on successful bootstrap
- `ServiceRegistered` - When new service comes online
```

## Advantages Over Manual Bootstrap

1. **Consistency**: Identical structure across all projects
2. **Speed**: Seconds vs minutes for setup
3. **Integration**: Direct Plane API integration, no manual copying
4. **Hydration**: All templates pre-filled with project metadata
5. **Error Reduction**: No manual UUID generation or typos

## Troubleshooting

### Missing Plane API Key
```bash
# Add to $ZC/secrets.zsh
export PLANE_API_KEY="your-api-key-here"
source $ZC/secrets.zsh
```

### pjangler Not Found
```bash
# Install/update pjangler
cd ~/code/pjangler
bun install
bun run build
```

### Plane API Errors
- Verify API key has workspace:write permissions
- Check network connectivity to Plane instance
- Ensure project identifier is unique in workspace
