# Platform Lifecycle (Cross-Component)

Use this only for platform-level coordination across multiple components.

## Typical triggers

- Cross-component feature or migration
- Platform readiness / maturity assessment
- Delegation across multiple teams/components
- Architecture-wide roadmap or sequencing

## Protocol

1. Define platform objective and affected components.
2. Break work into component-specific deliverables.
3. Sequence dependencies (parallel vs sequential).
4. Track status per component and integration gates.
5. Verify end-to-end system behavior.

## Required artifacts

- Cross-component story/plan
- Component assignment list
- Integration test checklist
- Rollout + rollback notes
